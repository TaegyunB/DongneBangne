# 1. 애자일
### 애자일이란?
변화에 유연하게 대응하며 빠르게 결과물을 만들어내는 소프트웨어 개발 방법론
<-> 폭포수(워터폴)

### 특징
- 짧은 주기로 개발
- 고객의 피드백을 즉시 반영
- 변화하는 상황에 맞춰 유동적으로 개발
- 계획과 회고가 1주일 단위, 스프린트로 돌아간다

### 스크럼  
: 애자일의 방법론 중 하나  
- 짧은 주기의 스프린트 단위로 사이클을 반복하며 개발
이번 프로젝트에서는 스크럼 방식 채택  
짧은 기간 결과물을 내는 것이 목표이기 때문!

# 2. JPA
### ORM 이란?
JDBC 프로그래밍의 복잡함이나 번거로움 없이 간단한 작업만으로 데이터베이스와 연동되는 시스템을 빠르게 개발할 수 있도록 기능을 제공하는 persistence framework.  

- 장점1. 직관적이고 가독성이 좋은 코드
- 장점2. 객체 지향적 접근으로 생산성 증가
- 장점3. 재사용 및 유지보수의 용이성 증가
- 장점4. DBMS에 대한 종속성 감소

- 단점1. 잘못 구현된 경우 속도 저하 발생
- 단점2. 복잡한 쿼리는 QueryDSL 등을 사용해 직접 처리해야 한다

JPA는 ORM(Object Relational Mapping) 기술에 해당한다. 

### JPA (Java Persistence API)
Java 어플리케이션에서 관계형 데이터베이스를 사용하는 방식을 정의한 인터페이스.

### MyBatis가 아닌 JPA여야 하는 이유
1. SQL 중심적인 개발에서 객체 중심으로 개발 가능
- CRUD 코드를 반복해야 되는 문제 개선
- 연관된 데이터 처리시 불편함 개선

2. 생산성을 높일 수 있음
- 수정이 매우 간단해짐
- 간단 CRUD 자동으로 할 수 있음

3. 유지보수 용이성
- 기존: 필드 변경 시 모든 SQL 수정
- JPA: 필드만 추가하면 됨. SQL은 JPA가 하니까 손댈 필요 없음
