## Docker
- 컨테이너를 활용하여 소프트웨어를 배포 및 구동하기 윈한 오픈 소스 플랫폼
- 특징
  - 운영체제를 가상화하지 않는 컨테이너 기술이니만큼 가상머신에 비해서 가벼우며, VM을 포함하여 한 대의 서버에 여러 개의 서비스를 구동하기 좋다
  - 보안상, 서비스가 털리더라도 원래의 서버에 영향을 미치기가 쉽지 않은 격리된 구조이므로 가상화의 장점을 상당 부분 활용 가능하다.
  - 다른 컨테이너의 환경을 쉽게 그대로 재현할 수 있는 이미지를 제공한다. 
    - 레이어로 구성되어 있으며 이를 유니온 마운트해서 관리
    - 다양한 미이지가 레이어를 공유할 수 있기 때문에 효율적으로 저장공간을 활용 가능
    - 이미지를 제공하는 원격 저장소가 있어서 누구나 쉽게 이미지 가져올 수 있다. 
- 현황
  - 구글, 아마존, 마이크로소프트에서도 도커를 지원 중
  - 사실상 업계 표준이니 만큼 사용자들이 작성해둔 소프트웨어 패키지/이미지들이 넘쳐나고 있어서 접근성과 사용하기 좋다는 장점이 있으며, 최근에는 클라우드 컴퓨팅에 대해 교육을 진행하는 교육기관에서도 도커에 대한 커리큠럼을 을 추가하는 경우가 증가되고 있다. 
  - 리눅스에서 유래한 기술이지만 원도우에서 리눅스 컨테이너를 구동하거나 윈도우 컨테이너를 구동할 수도 있다. 다만 리눅스 컨테이너 기술이니만큼 윈도우에서는 WSL2 안에, macOS에서는 VM 위에 올라가는데, 둘 다 가상머신을 이용하는 것이기에 리눅스와 비교하면 서능 저하가 있을 수 밖에 없다. 또한 윈도우의 경우 WSL2가 Hyper-v를 사용하기 때문에 VirutalBox나 VNware에서 VT-X를 사용할 수 없게 된다. 

- 기타
    - 컨테이너와 가상머신의 근본적인 차이
        - 컨테이너는 호스트 OS 커널을 공유하지만, 가상머신은 독립된 OS와 커널을 포함한다.
        - 컨테이너는 시작 속도가 빠르고, 리소스 사용량이 적으며, 수 초 이내로 부팅한다.
        - 가상머신은 전체 OS 부팅이 필요해 몇 분 걸릴 수 있고, 메모리와 디스크 사용량이 크다.
    - 컨테이너 기술의 DevOps/CI-CD 활용도
        - 컨테이너는 애플리케이션 실행 환경을 코드로 정의할 수 있어, CI/CD 파이프라인에서 반복 가능한 테스트 환경 제공에 유리하다.
        - 로컬 서버, 클라우드 환경의 차이를 최소화할 수 있어 개발과 운영 간의 환경 불일치 문제를 크게 줄인다. 
    - 컨테이너 오케스트레이션 시스템과의 관계
        - 오케스트레이션 도구는 대부분 리눅스 컨테이너를 기반으로 설계되어 있다.
        - 이에 윈도우나 macOS 환경에서 로컬 개발만 하고, 실제 운영은 리눅스 서버나 클라우드 상에서 수행하는 경우가 많다. 
    - 플랫폼에 따른 파일 시스템 차이
        - WSL2, macOS VM 기반 Docker는 호스트 - 컨테이너 간의 파일 시스템 접근 성능 이슈가 있을 수 있다. 
    - 컨테이너 보안과 격리 이슈
        - 컨테이너는 커널을 공유하기 때문에, 취약한 컨테이너를 통해 호스트 커널을 공격할 가능성이 있다.
        - 따라서 이를 보완하기 위해 seccomp, AppArmor, SELinux 등과 함께 사용하거나, gVisor, Kata Containers 등 경량 VM 기반 솔루션을 추가로 사용하기도 한다. 

- 단어 정리
    - 컨테이너
        - 정의: 애플리케이션과 그 실행에 필요한 모든 환경을 하나로 패키징한 가벼운 실행 단위
        - 특징
            - 호스트 OS의 커널을 공유(리눅스 기반)
            - VM보다 빠르게 실해되고, 리소스 사용량이 적음
            - 격리된 환경에서 실행되며, 시스템 간 이식성이 뛰어남
    - WSL2 (Windows Subsystem for Linux v2)
        - 정의: 윈도우 10/11에서 리눅스 환경을 실행할 수 있도록 지원하는 마이크로소프트의 서브시스템이다. 실제 리눅스 커널을 사용하는 경량 VM 기반이다. 
        - 특징
            - Docker Desktop이 리눅스 컨테이너 실행 시 내부적으로 WSL2를 사용
            - 윈도우와 리눅스 간 파일 고유 및 명령어 연동 가능
            - VirtualBox 등 일부 가상화 소프트웨어와 충돌 가능
    - 가상머신 (Virtual Machine, VM)
        - 정의: 하드웨어 위에서 전체 운영체저(OS)를 가상응로 실행할 수 있게 해주는 기술
        - 특징
            - Hypervisor가 하드웨어를 가상화해 OS 전체를 구동
            - 컨테이너보다 무겁고 느리지만, 커널까지 분리되어 보안과 격리성이 뛰어남
            - 서로 다른 OS도 실행 가능
    - Hypervisor (하이퍼바이저)
        - 정의: 가상머신을 구동하기 위한 핵심 소프트웨어. 물리적 자원을 가상 자원으로 분리하고, 각 VM에 할당
        - 특징
            - Type 1: 직접 하드웨어에서 실행
            - Type 2: 기존 OS 위에서 실행
