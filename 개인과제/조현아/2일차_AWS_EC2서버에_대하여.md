## EC2란?
- EC2는 Elastic Compute Cloud로 아마존 웹 서비스(AWS)에서 제공하는 클라우드 컴퓨팅 서비스이다.
- 클라우드 컴퓨팅이란, 인터넷을 통해 서버, 스토리지, 데이터베이스 등의 환경을 구축하는 것이다. 따라서 EC2는 AWS로부터 원격으로 제어할 수 있는 가상 컴퓨터를 한 대 빌리는 것이다.
- Elastic은, EC2가 후불제로 요금을 청구하기 때문이다. 사용한 만큼 비용을 지불하기 때문에 탄력적이라는 단어가 붙게되었다. 사용자는 성능과 용량을 조절하여 금액을 조절할 수 있다. 

## EC2를 사용해야 하는 이유
- 물리적인 서버 도입 시 장비 선택부터 설치까지 최소 1개월 이상의 시간이 소요된다. Vendor 선택, H/W 선택, 내부 품의, 장비 배송, 장비 입고, OS 설치 및 구성 등의 단계를 거쳐야 하기 때문이다.
- 반면에 AWS EC2를 도입하면 회원가입부터 설치까지 1시간이 채 걸리지 않는다. AWS 회원가입(30분), 서버 타입 및 사양 선택(10분), EC2 생성 및 구성(20분)

### 정리
- 효율성: 클릭 몇 번으로 서버를 생성할 수 있기 때문에 실제 서버를 구축하는 것보다 훨씬 간편하고 효율적이다.
- 비용 절감: 사용한 만큼만 요금을 지불하면 되므로 비용이 절감된다.
- 확장성: 트래픽이 증가하면 Auto Scaling을 통해 자동으로 인스턴스를 추가하고, 감소하면 제거할 수 있다.
- 다양한 운영체제 지원: Windows,Linux등 다양한 운영체제를 선택할 수 있다.
- 백업 및 복구: EBS 스냅샷을 통해 쉽게 백업하고 복구할 수 있다.
- 글로벌 서비스:전 세계 어디서든 서비스를 쉽게 제공할 수 있다. 

## EC2 인스턴스
- EC2를 사용하기 위해선 EC2 인스턴스를 생성해야 한다. 
- EC2 인스턴스를 생성한다는 것은 AMI를 토대로 운영체제, CPU, RAM 혹은 런타임이 구성된 컴퓨터를 빌리는 것이다. 
- 이때, 이 인스턴스는 사용목적에 따라 타입이 존재한다. 범용, 컴퓨팅 최적화, 메모리 최적화, 저장 최적화 등의 카테고리가 존재하며 각 카테고리당 4개의 타입이 존재하게 된다. 따라서, 프로젝트 별 최적의 성능을 지닌 서버를 구성할 수 있다.
- 특히, t와 m은 범용타입이기 때문에 초보자들이 가장 많이 사용한다.  
- EC2 인스턴스 사이즈는 인스턴스 cpu 갯수, 메모리 크리,성능 등이 있다.각 타입별로 nano,micro,small,medium,large,xlarge,2xlarge가 있으며 뒤로 갈수록 성능이 빨라진다. 
- EC2인스턴스는 "m5a.xlarge"이런 형식으로 쓰인다. 이때, m은 인스턴스 타입, 5는 5세대, a는 amd기반의 CPU 프로세서를 사용한다는 의미, xlarge는 인스턴스 사이즈를 의미한다. 

### 인스턴스 타입 상세
- 범용(General Purpose): t, m, a 시리즈 - 웹 서버, 소규모 DB에 적합
- 컴퓨팅 최적화(Compute Optimized): c 시리즈 - 고성능 웹 서버, 과학 연산에 적합
- 메모리 최적화(Memory Optimized): r, x, z 시리즈 - 실시간 빅데이터 처리, 인메모리 DB에 적합
- 저장 최적화(Storage Optimized): i, d, h 시리즈 - 분산 파일 시스템, 데이터 웨어하우징에 적합
- 가속화된 컴퓨팅(Accelerated Computing): p, g, f 시리즈 - 머신러닝, GPU 연산에 적합

## AMI란?
- Amazon 머신 이미지(AMI)는 소프트웨어 구성이 기재된 템플릿이다. 해당 템플렛에 EC2 인스턴스를 시작하는데 필요한 정보가 담겨있다.(운영체제, 애플리케이션 서버, 애플리케이션 등)
- AWS에서 제공하는 기본 AMI외에 사용자가 직접 생성한 Custom AMI를 사용할 수 있다.
