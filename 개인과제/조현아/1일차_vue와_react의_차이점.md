# 7/14일 개인과제
## Vue와 React의 차이점
### Vue란?
- Vue는 웹 애플리케이션의 사용자 인터페이스를 만들기 위해 사용하는 오픈소스 프로그레시브 자바스크립트 프레임워크이다. 
- Vue는 하나의 .vue 파일 내부에 HTML, CSS, Javascript를 작성할 수 있다는 장점이 있다. 따라서, 특정 기능을 담당하는 컴포넌트를 찾고 수정하기 수월하다.
- Vue는 러닝커브가 낮다. vue에서 제공하는 문법의 개수와 종류가 react에 비해 적어 빠른 시간 내 개발할 수 있다.
- vue는 라이브러리인 React와 달리 오픈소스 프레임워크이다. 따라서,부분적인 사용이 불가능하고, 프레임워크가 지원해주는 문법에 따라 작성해야한다. 따라서, 라이브러리에 비해 더 많은 기능을 기본으로 제공해준다. 
- vue는 새로운 컴포넌트를 분리하기 위해서는 새로운 파일을 하나 더 만들어야 한다는 단점이 있다. 
- vue는 템플릿 기반 구조를 사용하여 HTML과 유사한 문법으로 작성할 수 있어 기존 웹 개발자들이 익숙하게 접근할 수 있따.
- vue는 양방향 데이터 바인딩을 지원하여 v-model 디렉티브를 통해 폼 요소와 데이터를 쉽게 동기화할 수 있다. 

### React란?
- React는 자바스크립트 라이브러리의 하나로써 UI를 만들기 위해 사용된다. React는 라이브러리이기 때문에 React 자체만으로는 전역 상태 관리, 라우팅, 빌드 시스템 등을 지원하지 않는다. 따라서, React는 별도의 라이브러리를 통해 Redux, Recoil, React-router-dom 등을 사용해야한다. 따라서, 사용자가 필요할 때 필요한 부분 만을 가져다가 사용할 수 있다는 장점이 있다. 
- React는 JSX(JavaScript XML)형태로 코드를 작성한다. 해당 방법은 JavaScript 문법을 사용한다. 하지만, 이 형태의 학습이 어려워 진입장벽이 있다. 
- Vue는 지원해주는 문법에 한해서 개발이 가능하지만, React는 JavaScript의 문법을 사용하므로 개발자의 역량에 따라 자유롭게 사용 가능하다는 장점이 있다.
- React는 파일별로 컴포넌트를 분리할 수 있으며. 새로운 함수형 컴포넌트를 생산하고, prop으로 전달하는 것, 다른 곳에서 재사용 하는 것이 매우 용이하다. 
- React는 단방향 데이터 플로우를 따르며, 이는 애플리케이션의 데이터 흐름을 예측하기 쉽게 만든다.
- React는 컴포넌트 기반의 아키텍쳐를 통해 재사용성과 유지보수성을 높인다. 

### Vue와 React의 공통점
- 가상 DOM을 활용한다.
- 반응적이고 조합 가능한 컴포넌트를 제공한다.
- 컴포넌트 기반 아키텍쳐를 사용한다.
- 활발한 오픈소스 커뮤니티를 지니고 있다. 


### Vue와 React의 차이점 
#### 데이터 바인딩
- React는 단방향 데이터 바인딩을 사용하며, 데이터는 부모에서 자식으로만 흐른다.
- Vue는 양방향 데이터 바인딩을 지원하여 v-model을 통해 폼 요소와 데이터를 쉽게 동기화할 수 있다.

#### 성능
- Vue의 가상 DOM 구현이 일반적으로 React보다 빠르다고 알려져 있다.
- React는 더 큰 생태계와 최적화 도구들을 가지고 있어 대규모 애플리케이션에서 유리할 수 있다.

#### 개발 접근 방식
- React는 템플릿이 없으며 개발자에게 보다 많은 자율성을 제공한다. 하지만 JavaScript 문법에 능숙해야 한다.
- Vue는 보다 제한적인 기능을 제공하지만 러닝커브가 낮고, 많은 기능을 이미 구현해두어 활용하기가 보다 간편하다.

#### 생태계
- React는 더 큰 생태계와 더 많은 third-party 라이브러리를 가지고 있다.
- Vue는 공식적으로 관리되는 라이브러리들(Vue Router, Vuex/Pinia 등)을 통해 일관된 개발 경험을 제공한다.
