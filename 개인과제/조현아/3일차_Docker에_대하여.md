## Docker란?
- Docker는 컨테이너를 사용해 애플리케이션을 신속하게 구축, 테스트 및 배포할 수 있는 소프트웨어 플랫폼으로, 애플리케이션 및 그에 따른 환경을 격리된 컨테이너에 패키징하여 개발, 배포, 실행을 쉽게 지원하는 툴이다.
- Docker을 사용하면 컨테이너를 매우 가벼운 모듈식 가상머신처럼 다룰 수 있다. 또한 컨테이너를 구축, 배포, 복사하고, 한 환경에서 다른 환경으로 이동하는 등 유연하게 사용할 수 있어, 애플리케이션을 클라우드에 최적화하도록 지원한다.    
<br>  

## Docker의 특징 
### 1. 컨테이너 기반 가상화
- 컨테이너 가상화는 게스트 운영체제 없이 호스트 운영체제 위에 격리된 환경에서 가상화를 구현한다. 게스트 운영체제가 따로 존재하지 않으므로 호스트 운영체제의 요소를 공유하게 된다. 따라서 중복되는 요소가 줄어들어 성능이 개선된다. 

### 2. 가벼움과 빠른 실행 
- 가상환경을 사용하면 가벼워지며, VM을 포함하여 한 대의 서버에 여러 개의 서비스를 구동하기 좋다. 컨테이너는 ms단위로 실행되어 일반 프로세스를 실행하는 것과 큰 차이가 없다. 

### 3. 높은 시장 점유율
- Docker은 82%의 시장 점유율로 가장 널리 사용되는 컨테이너화 툴이다.

## Docker의 핵심 구성 요소
### Docker Engine
도커 엔진은 컨테이너를 생성하고 관리하는 주체로써 이 자체로도 컨테이너를 제어할 수 있고 다양한 기능을 제공하는 Docker의 핵심 프로젝트이다. 각 서버에 Docker Engine이 설치되어 컨테이너를 구축, 시작 또는 중단하는 데 사용할 수 있는 간단한 명령 세트를 제공한다.

### Docker Image 
Docker 이미지는 컨테이너를 정의하는 읽기 전용 템플릿이다. 이 이미지에는 코드에 필요한 라이브러리 및 종속성에 대한 정의를 비롯해 실행되는 코드가 포함되어 있다.

### Docker Container 
Docker 컨테이너는 인스턴스화된(실행되는) Docker 이미지다. 도커 컨테이너는 리눅스 기반의 컨테이너로, 인프라 또는 애플리케이션을 이루는 프로세스이며, 이러한 프로세스를 실행하는데 필요한 모든 파일이 고유한 이미지로 저장된다.
### Docker Hub
Docker Hub는 Docker 이미지의 공개 저장소로, 스스로를 컨테이너 이미지를 위한 세계 최대 라이브러리 및 커뮤니티라고 부른다. 상용 소프트웨어 공급업체, 오픈 소스 프로젝트 및 개별 개발자가 제공한 100,000개 이상의 컨테이너 이미지를 보유하고 있다.

도커는 빌드한 이미지를 서버에 배포하기 위해서, 직접 파일을 복사하는 방법 대신 도커 레지스트리(Docker Registry)를 이용한다. 도커 명령어를 이용하여, 이미지를 레지스트리에 push 하고 필요할 때는 레지스트리에서 pull하여 간편하게 사용 가능하다.

## Dockerfile

Dockerfile은 Docker 이미지(image)가 어떤 단계를 거쳐 빌드(build)되야 하는지를 담고있는 텍스트 파일이다. Docker는 Dockerfile에 나열된 명령문을 차례대로 수행하여 이미지를 생성해준다.

## Docker 사용의 장점
1. 환경 일관성
어떤 환경에서나 동일하게 동작하는 애플리케이션을 패키징할 수 있어 "내 컴퓨터에선 잘 돌아가는데...."의 문제를 해결할 수 있다. 

2. 빠른 배포
Docker을 사용하면 코드를 더 빨리 전달하고, 애플리케이션의 운영을 표준화하고, 코드를 원할하게 이동하고, 리소스 사용률을 높여 비용을 절감할 수 있다. 

3. 마이크로서비스 아키텍쳐 지원 
각 서비스를 독립적으로 컨테이너로 운영할 수 있어 마이크로서비스 아키텍쳐 구현에 적합하다.

4. DevOps 친화적
이 파일은 소스와 함께 버전 관리되고 원한다면 누구나 이미지 생성과정을 보고 수정할 수 있다.


<br>

## Docker 사용의 한계점
### 1. 커널 및 아키텍처 제약
호스트형 가상화 기술처럼 하드웨어를 연산으로 에뮬레이션하는 기술과 달리, 도커에서 사용되는 컨테이너형 가상화 기술은 호스트 운영 체제와 커널 리소스를 공유한다는 한계가 있다.

### 2. 데이터 지속성 문제
컨테이너는 삭제 후 새로 만들면 모든 데이터가 초기화된다. 그러므로 저장이 필요하다면, 업로드 파일을 외부 스토리지와 링크하여 사용하거나 S3같은 별도의 저장소가 필요하다.

### 3. 네트워크 복잡성
호스트 운영체제 레벨에서 네트워크를 한번 더 추상화 해야 해서 가상머신 보다 네트워크 구성에 신경을 써야 할게 많다.

### 4. 단일 컨테이너 관리 최적화
Docker는 단일 컨테이너 관리에 적합하도록 만들어져 있다는 특성이 있어, 여러 컨테이너를 관리할 때는 Docker Compose나 Kubernetes 같은 추가 도구가 필요하다.
